<Servlet 종속성 제거>
요청 파라미터 정보 자바의 Map 으로 넘김 -> 컨트롤러가 서블릿 기술을 몰라도 동작
request 객체를 Model 로 사용하는 대신에 별도의 Model 객체를 만들어서 반환

<View 이름 중복 제거>
컨트롤러는 뷰의 논리 이름을 반환, 실제 물리 위치의 이름은 프론트 컨트롤러에서 처리 -> 향후 뷰의 폴더 위치가 함께 이동해도 프론트 컨트롤러만 고치면 됨

프론트 컨트롤러가 기능이 있는 컨트롤러를 호출하면, 해당 컨트롤러는 modelView 객체를 반환한다
그리고 뷰의 논리 이름을 viewResolver 에서 물리 이름으로 변경하고, MyView 객체를 반환한다
이후 프론트 컨트롤러는 model 역할을 하는 메서드 render 를 호출하여 HTTP API(HTML 응답)을 할 것 이다
